# Plan: TypeScript Emit Support for Canopy

## Summary

Currently `promptToMarkdown()` is the only renderer â€” it always produces `---\nfrontmatter\n---\n## sections`. When `emitAs` ends in `.ts`, Canopy should instead emit a valid TypeScript module exporting the prompt body as a template-literal string constant.

## Architecture

The change is localized to **one file** (`src/commands/emit.ts`) with a small helper, plus tests. No type changes needed â€” `emitAs` already exists on the `Prompt` type and accepts any string.

### Key design decision

**Detect format by filename extension.** Every place that currently calls `promptToMarkdown()` will instead call a routing function that checks the filename extension and dispatches to the appropriate renderer. This keeps the change minimal and self-contained.

## Steps

### 1. Add `promptToTypeScript()` renderer in `src/commands/emit.ts`

Add a new function next to `promptToMarkdown()` (~lines 12-25):

```ts
function toExportName(promptName: string): string {
  return promptName.toUpperCase().replace(/-/g, "_");
}

function escapeTemplateLiteral(s: string): string {
  return s.replace(/\\/g, "\\\\").replace(/`/g, "\\`").replace(/\$\{/g, "\\${");
}

function promptToTypeScript(
  sections: { name: string; body: string }[],
  promptName: string,
  promptDescription?: string,
): string {
  const exportName = toExportName(promptName);
  const body = sections.map((s) => `## ${s.name}\n\n${s.body}`).join("\n\n");
  const escaped = escapeTemplateLiteral(body);

  const lines: string[] = [
    "// Auto-generated by Canopy â€” do not edit manually",
    `// Prompt: ${promptName}`,
  ];
  if (promptDescription) {
    lines.push(`// ${promptDescription}`);
  }
  lines.push("");
  lines.push(`export const ${exportName} = \`${escaped}\`;`);
  lines.push("");
  return lines.join("\n");
}
```

**Behavior details:**
- Export name: `day-planner-system` â†’ `DAY_PLANNER_SYSTEM` (uppercase, hyphens to underscores)
- Backticks escaped: `` ` `` â†’ `` \` ``
- `${` sequences escaped: `${foo}` â†’ `\${foo}`
- Backslashes escaped: `\n` literal in prompt body stays `\n` (not interpreted)
- Header comment: auto-generated notice + prompt name + optional description
- No frontmatter in TS output (frontmatter is markdown-specific metadata)
- Trailing newline for POSIX compliance

### 2. Add `renderPrompt()` dispatcher in `src/commands/emit.ts`

Add a function that routes based on filename extension:

```ts
function renderPrompt(
  filename: string,
  sections: { name: string; body: string }[],
  frontmatter: Record<string, unknown>,
  promptName: string,
  promptDescription?: string,
): string {
  if (filename.endsWith(".ts")) {
    return promptToTypeScript(sections, promptName, promptDescription);
  }
  return promptToMarkdown(sections, frontmatter, promptName, promptDescription);
}
```

## Edge cases handled

- Prompt body with no sections â†’ empty template literal
- Multiple sections â†’ joined with `\n\n` separator (same as markdown)
- Backslash in prompt body â†’ double-escaped so it survives template literal parsing
- Description field â†’ included in header comment if present
- Inheritance â†’ works unchanged (resolution happens before rendering)
